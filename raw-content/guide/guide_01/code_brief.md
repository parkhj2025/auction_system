## 지시

### 1. 본문 렌더

`post.md` 마크다운을 사이트 페이지 컴포넌트로 렌더한다.
- `[시각화 A]`, `[시각화 B]`, `[시각화 C]` 마커 위치에 해당 SVG를 삽입
- 각 시각화 위·아래에 충분한 여백 (사이트 spacing 토큰 기준)

### 2. 툴팁 인터랙션 11건

본문 내 백틱(`)으로 마킹된 11개 용어:

| # | 용어 | 정의 (1~2문장 권장) |
|---|------|-------------------|
| 1 | 강제경매 | 판결문·공정증서 등 집행권원이 있는 채권자가 신청하는 경매. 차용증으로 소송에서 승소한 후 그 판결문으로 신청하는 경우가 대표적. |
| 2 | 임의경매 | 등기된 담보권(근저당·전세권 등)을 근거로 신청하는 경매. 은행이 주택담보대출 시 설정한 근저당권을 근거로 신청하는 경우가 가장 흔하다. |
| 3 | 집행권원 | 강제집행을 할 수 있는 법적 근거가 되는 문서. 확정판결문, 공정증서, 화해조서 등이 해당된다. |
| 4 | 담보권 | 채권을 확보하기 위해 부동산 등에 설정하는 권리. 저당권·근저당권·전세권 등이 있다. |
| 5 | 근저당권 | 일정 한도(채권최고액) 안에서 채권을 담보하는 저당권. 은행이 대출 시 가장 일반적으로 설정하는 권리. |
| 6 | 최고가매수신고인 | 매각기일 입찰에서 가장 높은 가격을 적어낸 사람. 매각허가결정의 후보자가 된다. |
| 7 | 집행법원 | 경매 절차를 지휘하는 법원. 매각기일 지정, 매각허가결정, 대금지급기한 결정 등 주요 결정을 담당한다. |
| 8 | 집행관 | 법원의 위임을 받아 경매 현장을 진행하는 직책. 입찰 당일 경매법정에서 입찰을 받고 개찰을 수행한다. |
| 9 | 경매개시결정 | 채권자의 신청을 받아 법원이 경매 절차를 시작한다고 내리는 결정. 등기부에 기입되어 누구나 확인할 수 있다. |
| 10 | 배당요구 종기 | 다른 채권자들이 자기 몫을 신고할 수 있는 마감일. 이 날짜 이후의 채권은 배당에서 제외된다. |
| 11 | 인도명령 | 매수인이 잔금 납부 후 6개월 안에 신청하는 점유 회복 절차. 다만 대항력 있는 임차인 등은 대상에서 제외된다. |

구현:
- 마우스 호버 또는 모바일 탭 시 정의가 펼쳐지는 인터랙션
- 용어집 데이터는 별도 JSON 또는 TypeScript 객체로 관리 (다음 편들에서 추가될 용어 확장성 고려)
- 접근성: `aria-describedby` 또는 `<dfn>` 태그 활용

### 3. SVG 컬러 매핑

각 SVG 내부의 `c-{ramp}` 클래스를 사이트 디자인 토큰으로 매핑:

| SVG 클래스 | 의미 | 매핑 권장 |
|-----------|------|---------|
| `c-purple` | 1단계 준비 / 강제경매 | 사이트 보라 토큰 (Phase 8 컬러 토큰 18건 중) |
| `c-teal` | 2단계 매각 / 임의경매 | 사이트 청록 토큰 |
| `c-coral` | 3단계 결정 | 사이트 코랄 토큰 |
| `c-gray` | 4단계 인도 / 부동산 (중립) | 사이트 회색 토큰 |
| `c-blue` | 집행법원 / 집행관 (정보·권위) | 사이트 파란 토큰 |

각 ramp는 50/600/800 stop을 사용한다(fill / stroke / text). 다크모드는 사이트 토큰 시스템이 자동 처리하도록 매핑.

### 4. 면책고지 컴포넌트

페이지 하단의 3항목 면책고지(면책고지 / 업무범위 / 전문가 의뢰 권고)는 가이드 시리즈 모든 편에 동일하게 들어간다. 공통 컴포넌트로 분리할 것.

### 5. 다음 편 링크

마지막 섹션 "다음 편" 아래에 2편 페이지로의 링크 배치. 2편이 아직 없으면 비활성화 상태로 placeholder.

### 6. SEO 메타

- 제목: `경매란 무엇인가 — 경매 가이드 1편 | 경매퀵`
- 설명(약 150자): "경매가 처음이거나 큰 흐름은 알지만 세부가 헷갈리는 분을 위한 기본 정리. 강제경매와 임의경매의 차이, 다섯 등장인물의 역할, 경매 신청부터 인도까지 4단계 흐름을 시각 자료와 함께 정리합니다."
- OG 이미지: 시각화 C(4단계 흐름)를 1200×630 변형으로 활용 권장

### 7. 모바일 반응형

- SVG 3종 모두 viewBox 기반이므로 가로 폭 자동 조정
- 모바일(<480px)에서 시각화 텍스트 가독성 확인. 필요 시 viewBox 또는 폰트 크기 조정
- 툴팁 인터랙션 모바일 탭 동작 확인
