# 사법정보공유포털 / 공공데이터포털 경매정보 OpenAPI 신청 가이드

> **용도**: 대법원 경매정보(`courtauction.go.kr`)에 대한 비공식 WAF 우회 대신 정식 OpenAPI key로 데이터를 수급하기 위한 신청 절차.
> **목표**: 승인 시 Vercel 서버리스 포함 어느 IP에서든 안정적으로 대법원 사건 상세·사진 데이터를 가져올 수 있음. 옵션 D(Vercel 리전 변경) 및 옵션 A(자택 맥 CLI 재시드)가 전부 불필요해짐.
> **신청 주체**: 박형준 (경매퀵 대표, 공인중개사 자격 보유)
> **작성일**: 2026-04-24

---

## 1. 신청 채널 후보 (두 경로 중 하나 또는 둘 다)

### 1-A. 공공데이터포털 (data.go.kr) — 우선 시도 권장

**장점**: 신청 절차 간소, 온라인 즉시 제출, 승인까지 1~5영업일. 기관 구분 없이 개인·사업자 누구나 신청 가능.

**경로**:
1. https://www.data.go.kr 접속 → 회원가입 / 로그인
2. 검색창에 "**법원경매정보**", "**부동산 경매**", "**사법정보 공유**" 등 키워드 검색
3. 제공기관이 **법원행정처**인 데이터셋 필터링
4. 대상 데이터셋의 "활용신청" 클릭

**예상 제공기관**: 법원행정처 (대법원 산하 행정기관)

**검색 시 유의 키워드**:
- "법원 경매 물건"
- "부동산경매"
- "매각 물건 목록"
- "경매 사건"
- "사법 데이터"

실제 데이터셋 존재 여부와 정식 명칭은 **포털 검색으로 확인 필요**. 본 문서 작성 시점(2026-04-24) 기준 해당 데이터셋이 포털에 등록되어 있을 수도, 아닐 수도 있습니다.

### 1-B. 사법정보공유포털 (scourt.go.kr) — 공공데이터포털에 없을 경우

**장점**: 법원행정처가 직접 운영. API 정책 및 스키마가 가장 정확.

**예상 URL 후보**: `https://openapi.scourt.go.kr`, `https://www.scourt.go.kr/portal/openapi`, `https://data.scourt.go.kr`

**경로**:
1. 해당 포털 접속 → 회원가입 / 사업자 인증
2. 개발자 센터 / OpenAPI 신청 메뉴 진입
3. 신청서 작성 → 사용 목적·서비스 개요·예상 호출량 제출
4. 승인 대기 (법원행정처 내부 검토, 주~월 단위 소요 가능)

**현실성**: 공공데이터포털보다 절차가 무거울 수 있으나 데이터 품질·범위가 더 넓을 수 있음.

## 2. 신청서 작성 요령 (권장 문구 예시)

### 활용 목적란

```
경매 입찰 대리 서비스 "경매퀵"(공인중개사법에 따른 매수신청 대리)의
사건 매칭·시세 비교·콘텐츠 생성 파이프라인에서 대법원 공식 경매 정보를
근간 데이터로 활용하고자 합니다.

구체 사용처:
1. 고객 사건번호 입력 시 실시간 사건 조회 및 매칭 확정
2. 물건 목록·상세(입찰기일·감정가·최저가·소재지·면적)의 자동 수집
3. 물건 사진(전경·내부·매각물건 등)의 조회
4. 위 정보의 웹페이지(auctionquick.com 예정) 노출 및 고객 대면 콘텐츠 생성

상용 서비스이며 공인중개사법상 매수신청대리인 등록을 진행 중입니다.
대외 표기는 항상 "대법원 경매정보를 기초로 작성"을 준수합니다.
```

### 예상 호출량

```
일 호출량:
- 사건 목록 조회 (인천지방법원 기준): 1일 1회 배치, 약 500~2000건
- 사건 상세(사진 포함) 조회: 고객 신청 또는 콘텐츠 생성 시 온디맨드,
  일 평균 50회 내외, 최대 300회.
- 향후 수원·대전·부산·대구 확장 시 상기 건수의 3~5배.

피크 시간대: 평일 09:00~18:00
Rate limit 준수: 초당 1회 이하, 필요 시 협의.
```

### 연락처·증빙

- 사업자등록번호 (등록 완료 후 기재)
- 공인중개사 자격증 사본 (박형준)
- 매수신청대리인 등록증 사본 (등록 완료 후)
- 서비스 도메인 (auctionquick.com 등 확정 후)
- 담당자 이메일·연락처

> **주의**: 현재 사업자등록 진행 중이라면 신청서 제출을 **등록 완료 이후**로 연기하는 것이 승인 확률을 높일 수 있습니다.

### 개인정보·보안 정책

- 수집 데이터는 경매 공공정보 한정, 개인 식별정보 수집 없음
- API key는 서버 환경변수로 관리, 클라이언트에 노출 금지
- Supabase(AWS Seoul region) 상에서 is_active·raw_snapshot 형태로 보존
- 대외 재배포 시 출처 표기 의무 준수

## 3. 승인 후 확인·반영 사항

### 3-1. API 응답 스키마 대조

공공 OpenAPI 응답 필드가 기존 비공식 경로(`searchControllerMain.on` / `selectAuctnCsSrchRslt.on`)와 다를 가능성이 큽니다. 다음을 대조:

- 사건 식별자 포맷 (현재 `docid`가 이 API에서도 그대로 나오는지)
- `csPicLst` 또는 유사 사진 배열 필드명
- base64 vs URL 방식 (OpenAPI는 이미지 URL 직접 제공일 가능성 높음)
- 카테고리 코드 (`000241`~`000247`) 유지 여부
- Rate limit (일/월/시간당)

### 3-2. 웹사이트 연동 포인트 (승인 후 작업 예상)

| 파일 | 변경 내용 |
|------|-----------|
| `src/lib/courtAuction/codes.ts` | `BASE_URL`, `DETAIL_ENDPOINT`, 헤더 세트 교체 |
| `src/lib/courtAuction/session.ts` | 세션 2단계 초기화 → API key 헤더만 추가로 단순화 |
| `src/lib/courtAuction/photos.ts` | `csPicLst` 스키마 대조 후 필드 매핑 조정 |
| `scripts/crawler/api.mjs` | 크롤러 목록 API 교체 |
| `.env.local` / Vercel env | `COURTAUCTION_API_KEY` 신규 추가 |
| `docs/courtauction-photo-fetch.md` | recipe 개정 (v2로 버전 bump) |

작업 분량: 반나절~1일 예상 (스키마 대조 완료 시).

### 3-3. WAF 우회 로직 제거

승인 반영이 완료되면 다음 회피 코드를 정리:

- `vercel.json`의 `functions.regions` 설정 (sin1 등 임시 리전)
- `photos.ts`의 "빈 csPicLst 방어" / "재fetch 부족 시 기존 캐시 유지" 로직 (부분 유지는 가능)
- `scripts/seed-photos.mjs` CLI (레거시로 남기되 기본 워크플로우에서는 미사용)
- 자택 IP 의존 운영 메모

## 5. 병행 운영 방침 (승인 대기 중)

승인까지 최소 1영업일~최대 수주 소요 예상. 그 동안은 다음 조합으로 운영:

- **1차 방어**: 옵션 D (Vercel 리전 `sin1` 등 변경) — 성공 시 대부분 자동 해결
- **2차 방어**: 옵션 A ([scripts/seed-photos.mjs](../scripts/seed-photos.mjs)) — 신규 사건 투입 전 맥에서 1회 실행
- **3차 방어**: `photos.ts` 내장 방어 로직 (빈 응답 시 기존 캐시 유지) — 자동 작동

세 방어가 모두 발동하더라도 OpenAPI 승인이 근본 해결이 되므로 신청은 가능한 한 빠른 시점에 제출 권장합니다.

## 6. 문의 대응 예상 Q&A (법원행정처가 물을 수 있는 질문)

| 질문 | 준비된 답변 |
|------|-------------|
| 상용 서비스인가? | 예. 공인중개사법상 매수신청 대리 업무 수행. 수수료 모델 존재. |
| 데이터를 재가공해 판매하는가? | 아니요. 내부 매칭·고객 안내·자사 블로그 콘텐츠 한정. 원본 재판매·대량 배포 없음. |
| 크롤링을 별도로 하고 있는가? | 현재 임시로 비공식 경로 사용. OpenAPI 승인 후 즉시 전환 예정. |
| 기대 월간 호출량은? | 목록 배치 30회 + 상세·사진 3,000회 내외. 확장 시 월 20,000회 이하. |
| 보관 기간·삭제 정책은? | `is_active=false`로 소프트 비활성화. 원본 `raw_snapshot` JSONB 영구 보존(감사 목적). 고객 대면 노출은 `is_active=true`만. |
