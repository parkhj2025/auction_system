## v2 업그레이드 범위

### 1. PG사 연동 (카드·간편결제)

#### 선정 후보

| PG사 | 월 고정비 (추정) | 수수료율 (카드) | 간편결제 지원 | 연동 난이도 | 비고 |
|---|---|---|---|---|---|
| 토스페이먼츠 | 0원 (건당 수수료만) | 2.9% | 토스페이·카카오페이·네이버페이 | 낮음 (REST API 단순) | 스타트업 친화적. 정산 주기 D+2 |
| 나이스페이먼츠 | 30,000~50,000원 | 2.4~2.9% | 카카오페이·네이버페이·페이코 | 중간 | 업력 긴 안정성. 국내 MAU 1위 |
| KG이니시스 | 30,000~50,000원 | 2.9% | 전체 간편결제 커버 | 중간 | 대형 쇼핑몰 표준 |
| 카카오페이 직연동 | 0원 | 2.9% | 카카오페이만 | 낮음 | 단일 채널 한정 |

#### 선정 기준

1. **월 고정비 최소화**: Phase 2 초기 140건/월 수준에서 고정비 대비 건당 원가 계산.
2. **간편결제 커버리지**: 30~50대 직장인 주 고객층 → 카카오페이·네이버페이 필수.
3. **연동·운영 공수**: 작은 팀에 API가 단순할수록 유리.
4. **정산 주기·환불 처리**: 패찰 시 보증금 전액 반환 프로세스와 충돌 없어야 함.

**추천 기본안** (확정 시점에 재검토): **토스페이먼츠**. 월 고정비 0 + 주요 간편결제 전 커버 + REST API 단순.

#### 연동 가이드 뼈대

- `/api/checkout/create` — 주문 생성 + PG 결제창 URL 발급.
- `/api/checkout/webhook` — 결제 완료/실패 콜백 처리. 주문 상태 전환.
- `/api/checkout/refund` — 패찰 시 보증금 반환 (결제 취소 또는 부분 환불).
- DB 스키마 확장: `orders.payment_provider`, `orders.payment_tx_id`, `orders.payment_status`.
- 기존 계좌이체 흐름은 v2에서도 보조 옵션으로 유지 (PG 결제 실패·대체 수단).

### 2. 본인인증 실 SDK

#### 선정 후보

| SDK | 월 고정비 (추정) | 건당 비용 | 통신사 커버 | 연동 난이도 | 비고 |
|---|---|---|---|---|---|
| NICE 본인확인 | 20,000~50,000원 | 80~150원 | SKT·KT·LGU+ 전체 | 중간 (팝업형) | 시장 점유율 1위. 표준 선택 |
| KCB | 30,000~50,000원 | 80~150원 | 전체 | 중간 | 금융권 친화. 정보 정합성 강함 |
| 다날 | 20,000~40,000원 | 100~150원 | 전체 | 낮음 (REST API) | 간편 연동 |
| 토스 본인확인 | 0원 (건당만) | 120~200원 | 전체 | 낮음 | 신규. 향후 개선 여지 |

#### 선정 기준

1. **통신사 전체 커버**: 3사 전부 지원 필수.
2. **연동 난이도**: Mock SDK → 실 SDK 교체 공수가 작을수록 유리.
3. **건당 비용**: 월 140건 × 건당 100원 = 월 14,000원. 고정비가 더 큰 비중.
4. **결제 SDK와 번들**: PG사와 동일 벤더 묶음 시 할인 여지 (예: 나이스페이먼츠 + NICE 본인확인).

**추천 기본안**: **NICE 본인확인**. 시장 표준 + 안정성 + PG 선택과 독립적.

#### 연동 가이드 뼈대

- 현재 Mock SDK 인터페이스 `verifyPhone()` 시그니처 유지 → 내부 구현만 실 SDK로 교체.
- 기존 `/apply` Step 본인인증 UI는 그대로, 호출 모듈만 변경.
- DB 스키마: `orders.verified_phone`, `orders.verified_at`, `orders.verification_ci` (CI 저장 필요 시) 추가 검토.

### 3. 카카오톡 알림톡 (자동화)

#### 선정 후보

| 발송 플랫폼 | 월 고정비 | 건당 비용 | 템플릿 승인 공수 | 연동 난이도 | 비고 |
|---|---|---|---|---|---|
| 카카오 비즈메시지 직 API | 0원 | 8~12원 (알림톡) | 직접 제출 | 높음 (심사 3~7일) | 최저 단가 |
| 솔라피 (Solapi) | 0원 | 9~13원 | 대행 | 낮음 (REST API) | 한국 표준 SMS/알림톡 Aggregator |
| 루나소프트·알리고 | 0원 | 10~15원 | 대행 | 낮음 | 국내 다수 이용 |

#### 선정 기준

1. **템플릿 승인 경험**: 심사 반려 대응 공수가 관건. Aggregator가 유리.
2. **SMS 대체 발송**: 알림톡 실패 시 SMS 자동 fallback 지원 여부.
3. **발송 이력 조회**: 운영 대시보드에서 건별 발송 성공 확인 가능 여부.

**추천 기본안**: **솔라피**. REST API 단순 + 템플릿 대행 + SMS fallback.

#### 알림톡 템플릿 (초안)

1. **접수 확인**: "[경매퀵] 접수되었습니다. 사건번호 {case_no} {round}회차. 보증금 입금 후 확정됩니다."
2. **보증금 확인**: "[경매퀵] 보증금 입금이 확인되었습니다. 입찰일까지 대기해 주세요."
3. **입찰 수행 완료**: "[경매퀵] 입찰이 완료되었습니다. 결과는 {result_date} 이후 통보됩니다."
4. **결과 통보 (낙찰)**: "[경매퀵] 낙찰되었습니다. 성공보수 정산 안내를 곧 드립니다."
5. **결과 통보 (패찰)**: "[경매퀵] 패찰되었습니다. 보증금은 영업일 기준 2일 이내 전액 반환됩니다."

#### 연동 가이드 뼈대

- `src/lib/notify/kakao.ts` 모듈 신설. 주문 상태 전환 시점(접수/입금/입찰/결과)마다 호출.
- DB 트리거 또는 API 레이어 hook에서 발송.
- 발송 이력 테이블: `notifications(order_id, template_key, sent_at, status, response)`.

## v2 총 예상 월 비용 (참고 추정)

| 영역 | 월 고정비 | 월 변동비 (140건 기준) | 소계 |
|---|---|---|---|

**주의**: 위 수치는 2026-04-21 기준 공개 정보의 대략적 추정. v2 진입 시점에 각 벤더의 최신 요금제를 재조사하여 확정.

## v2 진입 조건 (형준님 판단 사항)
